2.Like 5:Что такое DNS (Domain Name System) и в чём разница протоколов HTTP/HTTPS и их версий -краткое эссе, своими словамиDNS-это система доменных имен. 

Домен-это онлайн адрес сайта.У каждого сайта есть свой IP-адрес, состоящий  из разнообразных комбинацийчисел разделенных точками. 

Чтобы упростить для пользователя задачу запоминания адреса сайта, придумали текстовые( доменные)адреса. 

Поэтому текстовый адрес сайта и IPадрес сайта-это один и тот же адрес. 
Часть домена, что расположена справа от точки –это домен верхневого уровня. ( TLD) Они делятся на 2 группы: национальные-например,ru. 
Тематические-.com(коммерция).Слева от последней точки-домен 2 уровня. (например, site.com). Слева же от домена 2 уровня располагается домен 3 уровня. (например, blog.site.com)

DNS сервер поддерживает (обслуживает) доменные зоны, которые ему делегированы. Он непосредственно хранит данные о ресурсных записях для зоны. 
Например, что сервер, на котором находится сайт "example.ru", имеет IP адрес "1.1.1.1". DNS сервер отвечает на все запросы, касательной этих доменных зон. 
Если ему приходит запрос о домене, который ему не делегирован, то он спрашивает ответ у других DNS серверов.


HTTPS (Hyper Text Transfer Protocol) и HTTP протоколы передачи гипер-текста, предназначенные для обмена данными на сайтах.
Основаны на технологии "клиент-сервер".

В HTTP информация передается в незашифрованном виде. ( браузер  выдает "подключение к этому сайту не защищено)

В HTTPS (буква "S" означает secure - безопасный протокол) в нем используется шифрование с помощью криптографического протоколов "SSL" и "TLS".
Ключи шифрования хранятся на сервере, для расшифровки используется SSL-сертификат.
Протокол HTTPS важен для сайтов с онлайн-платежами, интернет-магазинов, банковских сайтов и т.п.

Основные отличия: 
HTTP:
- порт 80
- отображение в браузере о небезопасном подлючении.
HTTPS:
- порт 443
- надстройка шифрования ( нужны ключи)
- сохранность данных
- безопасный протокол
 
Существует несколько версий протоколов HTTP. Самая ранняя версия протокола -это HTTP-0.9.Впервые была опуюликована в 1992 году.

Спецификация протокола привела к упорядочению правил взаимодействия между клиентами и серверами HTTP, а также чёткому разделению функций между
этими двумя компонентами. Были задокументированы основные синтаксические и семантические положения.

Следующая версия была выпущена в 1996 году -это HTTP-1.Более современная его версия — HTTP/1.1 —  появилась в 1999 году. 
 
Этот протокол  научился поддерживать режим постоянного соединения, т.е. оставлять соединение открытым после того как получен ответ на запрос. 
Это позволило за одно соединение посылать сразу несколько запросов, а не открывать-закрывать сессию каждый раз.

В 2015 году вышла новая версия - протокол HTTP-2. В отличии от предыдущего протокола, он бинарный , т.е использует двоичную уровень кадрирования.
Двоичный уровень кадрирования организует сообщения в параллельные потоки данных, что что позволяет существенно ускорить и оптимизировать загрузку содержимого сайта.
Это особенно важно при работе с графикой, видео,мобильными платформами.

Основные его особенности:

1.Расстановка и управление приоритетами запросов/потоков — клиент самостоятельно задает для сервера приоритетность ресурсов и данных

2.Сжатие HTTP заголовков.

3.Мультиплексирование запросов или параллельная загрузка по TCP-соединению нескольких элементов сайта — через одно соединение отправляется несколько запросов, 
а ответы клиент может получать в любом порядке.

4.Наличие и поддержка со стороны сервера проактивных push-уведомлений — сервер самостоятельно может отправлять данные для клиента, которые тот еще не запросил.
